Title: Backup Integrity Verification & Upload (Ops Finalization)

Goal: Upload at least one encrypted backup object to the GCS bucket and run a decrypt-integrity test to confirm backup system health.

Status: Ready for Ops

Preconditions:
- GCS bucket `gs://gcp-eiq-ci-artifacts` has been created and is accessible.
- At least one encrypted backup file exists (or will be generated by CI `maintenance:gitlab_backup` job).

Checklist:

- [ ] Confirm GCS bucket is accessible and empty:
  ```bash
  gsutil ls -l gs://gcp-eiq-ci-artifacts/** || echo "No objects yet"
  ```
- [ ] Upload a sample encrypted backup (if no backups exist yet):
  ```bash
  # From a machine with the backup file or CI artifact
  gsutil cp ./gitlab-backup-20260306.tar.age gs://gcp-eiq-ci-artifacts/backups/
  ```
- [ ] Retrieve the `age` decryption private key (stored securely; location TBD):
  ```bash
  # Confirm you have access to the age key used to encrypt backups
  ls -la ~/.age/key.txt  # or wherever the key is stored
  ```
- [ ] Download and decrypt a sample backup:
  ```bash
  gsutil cp gs://gcp-eiq-ci-artifacts/backups/gitlab-backup-*.tar.age /tmp/
  age -d -i ~/.age/key.txt /tmp/gitlab-backup-*.tar.age > /tmp/gitlab-backup.tar
  ```
- [ ] Verify archive integrity:
  ```bash
  tar -tzf /tmp/gitlab-backup.tar | head -20
  echo "✓ Backup archive is valid and decrypts successfully."
  ```
- [ ] Confirm `/etc/gitlab/gitlab-secrets.json` and `/etc/gitlab/gitlab.rb` are present in the backup (if applicable):
  ```bash
  tar -tzf /tmp/gitlab-backup.tar | grep -E "(gitlab-secrets|gitlab\.rb)" || echo "Not found in sample"
  ```
- [ ] Clean up temporary files:
  ```bash
  rm -f /tmp/gitlab-backup* 
  ```
- [ ] Close this issue once verification is complete.

Reference:
- Ops Finalization Runbook: `docs/OPS_FINALIZATION_RUNBOOK.md` (Step 5)
- Backup script: `scripts/backup/gitlab_backup_encrypt.sh`
- Restore script: `bootstrap/restore_from_github.sh`

Notes:
- Backups are encrypted with the `age` tool; ensure the private key is stored securely and backed up in multiple locations.
- The `ci-gcs-bucket` secret stores the target bucket name (`gs://gcp-eiq-ci-artifacts`).
- Backup integrity tests are non-destructive and use a temporary copy.
- A full live DR dry-run (issues/903, 905) will validate the complete restore flow including decryption and project import.

Completion Criteria:
- A sample encrypted backup object exists in `gs://gcp-eiq-ci-artifacts/`.
- The backup decrypts successfully using the `age` private key.
- The decrypted archive is valid and contains expected GitLab backup objects.
