**STATUS:** ✅ MANDATORY ACROSS ALL SYSTEMS


**ENFORCEMENT:**

### File Organization Decision Tree

**When creating a new file, ask:**

1. Is it **production code**?
   - Go to: `api/`, `backend/`, `frontend/`, `nexusshield/`, or `infra/`

2. Is it a **script**?
   - Ask: What does it do?
     - Deploys? → `scripts/deployment/`
     - Provisions? → `scripts/provisioning/`
     - Orchestrates? → `scripts/automation/`
     - Utility? → `scripts/utilities/`

3. Is it **documentation**?
   - Ask: What type?
     - Governance/Policy? → `docs/governance/`
     - Deployment/Setup? → `docs/deployment/`
     - Operational runbook? → `docs/runbooks/`
     - Architecture? → `docs/architecture/`
     - Historical? → `docs/archive/`

4. Is it a **config file**?
   - Go to: `config/`

5. Is it **old/legacy**?
   - Go to: `docs/archive/` (keep forever, immutable)

6. Is it a **test**?
   - Go to: `tests/`

7. Is it **monitoring/observability**?
   - Go to: `monitoring/`

## 🔐 Governance Standards

### Naming Conventions

✅ **Approved:**
- Lowercase folder names: `docs/`, `scripts/deployment/`
- Hyphenated file names: `elite-setup.sh`, `governance-standards.md`
- Descriptive names: `deploy-to-production.sh` (not `deploy.sh`)
- Date format: `YYYY-MM-DD` (if needed): `2026-03-10-report.md`

❌ **Forbidden:**
- Mixed case folders: `Docs/`, `Scripts/`
- Spaces in filenames: `My File.md` (use `my-file.md`)
- Undescriptive names: `script.sh`, `file.md`
- Duplicate files: Don't keep both `old_deploy.sh` and `deploy.sh`
- Random dates in filenames: avoid `DEPLOYMENT_FINAL_STATUS_2026_03_09.md`

### Documentation Requirements

Every major directory must include:

- **README.md** - Explains folder purpose and contents
- **STRUCTURE.md** - (if complex) Describes subdirectory layout
- **INDEX.md** - (if >20 files) Lists and indexes contents

Example:
```
scripts/deployment/
├── README.md          # "Deployment automation scripts"
├── deploy-to-prod.sh  # Actual script
└── deploy-to-staging.sh
```

### Git Commit Standards

When organizing/moving files:

```bash
# Good commits
git commit -m "refactor: move deployment scripts to scripts/deployment/"
git commit -m "docs: consolidate governance standards to docs/governance/"

# Bad commits
git commit -m "cleanup"
git commit -m "moved stuff"
```

## 🔄 Maintenance Procedures

### Monthly Organization Review
1. Check for files in root: `find . -maxdepth 1 -type f | grep -E "\.(md|sh|py|yml)$"`
2. Check for 6+ deep directories: `find . -type d -depth +5`
3. Review logs/ for old files (>30 days)
4. Verify no duplicates: `find . -type f -name "*.md" | sort | uniq -d`

### Archival Procedure
1. Move historical files to `docs/archive/`
2. Add timestamp prefix: `2026-03-10-filename.md`
3. Keep immutable (never delete from archive)
4. Index in `docs/archive/INDEX.md`

### New Team Member Onboarding
1. Point to `FOLDER_STRUCTURE.md`
2. Show example: "Code goes in api/backend/frontend/, Docs go in docs/, Scripts go in scripts/"
3. Test: Ask them to create a sample file in correct location
