6. Is it a **test**?

## 🔐 Governance Standards

### Naming Conventions

✅ **Approved:**
- Lowercase folder names: `docs/`, `scripts/deployment/`
- Hyphenated file names: `elite-setup.sh`, `governance-standards.md`
- Descriptive names: `deploy-to-production.sh` (not `deploy.sh`)
- Date format: `YYYY-MM-DD` (if needed): `2026-03-10-report.md`

❌ **Forbidden:**
- Mixed case folders: `Docs/`, `Scripts/`
- Spaces in filenames: `My File.md` (use `my-file.md`)
- Undescriptive names: `script.sh`, `file.md`
- Duplicate files: Don't keep both `old_deploy.sh` and `deploy.sh`
- Random dates in filenames: avoid `DEPLOYMENT_FINAL_STATUS_2026_03_09.md`

### Documentation Requirements

Every major directory must include:

- **README.md** - Explains folder purpose and contents
- **STRUCTURE.md** - (if complex) Describes subdirectory layout
- **INDEX.md** - (if >20 files) Lists and indexes contents

Example:
```
scripts/deployment/
├── README.md          # "Deployment automation scripts"
├── deploy-to-prod.sh  # Actual script
└── deploy-to-staging.sh
```

### Git Commit Standards

## 🔧 TROUBLESHOOTING GUIDE

### SSH Key Issues

**Problem:** SSH connection fails with "Permission denied"  
**Solution:** Verify key format and permissions:
```bash
# Check key format (should be RFC4716)
file ~/.ssh/id_ed25519
# Should output: ...OpenSSH private key format...

# Verify key permissions
ls -la ~/.ssh/id_ed25519
# Should be: -rw------- (600)

# Test connection with verbose output
ssh -vvv -i /path/to/key user@host echo "Connected"
```

**Problem:** Keys not found in GSM  
**Solution:** Verify GSM integration:
```bash
# List all GSM secrets
gcloud secrets list --project=$PROJECT_ID

# Retrieve specific key
gcloud secrets versions access latest --secret="ssh-key-account-name" --project=$PROJECT_ID
```

### Health Check Issues

**Problem:** Health checks showing FAIL  
**Solution:** Check systemd service status:
```bash
# View service status
systemctl --user status ssh-health-checks.service

# View recent logs
journalctl --user -u ssh-health-checks.service -n 20

# Manual test
bash scripts/ssh_service_accounts/health_check.sh --verbose
```

### Rotation Issues

**Problem:** Credential rotation failing  
**Solution:** Verify rotation configuration:
```bash
# Check rotation schedule
systemctl --user list-timers credential-rotation.timer

# View rotation logs
journalctl --user -u credential-rotation.service -n 50

# Manual rotation test
bash scripts/ssh_service_accounts/credential_rotation.sh --dry-run
```

### Audit Trail Issues

**Problem:** Audit logs not recording  
**Solution:** Verify JSONL logging:
```bash
# Check audit trail
cat audit-trail.jsonl | tail -20

# Validate JSONL format
jq '.' audit-trail.jsonl | head -20

# Check disk space
du -sh audit-trail.jsonl
```

### Quick Diagnostic Commands

```bash
# Full system health check
bash scripts/ssh_service_accounts/health_check.sh

# Verify all 32 accounts
```
