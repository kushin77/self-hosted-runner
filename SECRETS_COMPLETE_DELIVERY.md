## 🔐 Security Reminders

1. **Never commit secrets** to git
   ```bash
   # Good
   gh secret set RUNNER_MGMT_TOKEN --repo ... --body "$TOKEN"
   
   # Bad ❌
   echo "ghp_xxxx" > .env && git add .env
   ```

2. **Protect your PAT**
   - Don't share in chat/email
   - Use GitHub Secrets for storage
   - Rotate every 90 days

3. **SSH Key Security**
   - Generate with no passphrase (GitHub can't handle interactive prompts)
   - Store public key on runner hosts only
   - Private key only in GitHub Secrets

4. **Audit Trail**
   - All automation runs logged in GitHub Actions
   - All cleanup operations timestamped
   - Monthly validation reports created as issues

## 📞 Support Resources

**If something fails:**

1. **Check logs**: `gh run view <RUN_ID> --repo kushin77/self-hosted-runner --log`
2. **Review guide**: `SECRETS_SETUP_GUIDE.md` → Troubleshooting section
3. **Run validation**: `bash scripts/automation/validate-idempotency.sh`
4. **Manual test**: `ssh -i ~/.ssh/runner_key runner@host echo OK`

**Common Issues & Fixes:**

| Issue | Solution |
|-------|----------|
| "gh: command not found" | Install: `apt-get install gh` |
| "Not authenticated" | Run: `gh auth login` |
| "SSH key rejected" | Check public key on runner host: `grep YOUR_KEY ~/.ssh/authorized_keys` |
| "Token invalid" | Regenerate at https://github.com/settings/tokens, re-set with `gh secret set` |
