## 📋 What's Been Created

### 1. **Interactive Setup Script**
📄 **File:** [scripts/setup-azure-tenant-api.sh](../scripts/setup-azure-tenant-api.sh)  
**Size:** ~26KB

## 🔐 Security

### ✅ What's Secure:
- ✅ **No hardcoded secrets** - All stored externally (GSM/Vault)
- ✅ **No Git exposure** - Secrets NEVER in repository
- ✅ **Audit trail** - Immutable JSONL logs of all operations
- ✅ **Multi-layer fallback** - GSM → Vault → Environment
- ✅ **Runtime injection** - Secrets fetched at execution time
- ✅ **Service principal** - Never uses personal accounts for automation
- ✅ **90-day rotation** - Documented rotation procedures
- ✅ **Least privilege** - Scoped to subscription + specific roles

### ⚠️ Important Notes:
- **Client secret is sensitive** - Only stored in GSM/Vault (never echo it)
- **90-day rotation** - Script creates credentials; plan to rotate in 90 days
- **Service principal = subscription owner** - Has broad permissions; monitor access logs
- **Activity logs** - All Azure operations logged and auditable

## ❓ FAQ

**Q: Do I need to commit anything to Git?**  
A: No! The credentials are stored in GSM/Vault (external). Only the helper scripts are committed.

**Q: Can I delete/reset the service principal later?**  
A: Yes. Run: `az ad sp delete --id "$AZURE_CLIENT_ID"`

**Q: What if I lose the credentials?**  
A: They're stored in GSM/Vault. Just fetch them again: `gcloud secrets versions access latest --secret="azure-client-id"`

**Q: How do I rotate credentials after 90 days?**  
A: See: `docs/AZURE_TENANT_API_SETUP.md` → "Credential Rotation" section

**Q: Can I use this with other Azure services?**  
A: Yes! Service principal works with: Storage, Databases, VMs, App Services, Key Vaults, etc.

**Q: What if Azure CLI isn't installed?**  
A: Setup script installs it automatically (Ubuntu/Debian only; for macOS, use `brew install azure-cli`)

## 📞 Troubleshooting

If something goes wrong:

1. **Check logs:**
   ```bash
   cat /tmp/azure-setup-*.log
   tail logs/azure-setup/*.jsonl | jq .
   ```

2. **Verify installation:**
   ```bash
   az --version
   gcloud --version
   vault --version 2>/dev/null || echo "Vault optional"
   ```

3. **Test credentials manually:**
   ```bash
   az login --service-principal \
     -u "$AZURE_CLIENT_ID" \
     -p "$AZURE_CLIENT_SECRET" \
     --tenant "$AZURE_TENANT_ID"
   ```

4. **Check GSM access:**
   ```bash
   gcloud secrets list --filter="name:azure-*"
   ```

See full troubleshooting: `docs/AZURE_TENANT_API_SETUP.md`
