# Credential Rotation Log

Use this file to record credential rotations performed by maintainers following the remediation initiative. Each entry should include date, actor, credential/service, and verification status.

Example:

- 2026-03-07 — @kushin77 — GitHub deploy key `deploy-runner` — rotated, verified
- 2026-03-07 — @kushin77 — Vault `app/config` — rotated, apps verified

Follow `CREDENTIAL_ROTATION_CHECKLIST.md` for the rotation procedure.
